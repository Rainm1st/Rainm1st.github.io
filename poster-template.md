# TrainerSync Poster Template (Week 8)

## Title
TrainerSync: Real-time Coach-Trainee Interaction Loop for Onsite Fitness Training

## 1. Background & Motivation
- Multi-user onsite sessions make it hard for coaches to provide timely guidance to every trainee.
- Trainees often receive fragmented instructions and lack quality-focused post-session feedback.
- We aim to build a human-centric web app that supports real-time intervention and motivational continuity.

## 2. Aims & Objectives
- Design a dual-interface system for trainees and coaches.
- Provide real-time training cues using heart-rate zone feedback.
- Build a closed loop from task execution to coach intervention to post-session recap.

## 3. User Research Summary
- Stakeholders: gym trainees, trainers/coaches.
- Methods: interviews/questionnaires + literature review + competitor analysis.
- Key findings:
  - Trainees need actionable in-session prompts.
  - Coaches need low-friction multi-user monitoring and one-tap interventions.
  - Motivation is strengthened by lightweight gamification.

## 4. Personas
- Persona A: beginner trainee (mobile-first, feedback-driven).
- Persona B: coach (high-pressure context, rapid decision-making).

## 5. User Journey
- Before session -> warm-up -> main workout -> coach intervention -> cooldown/recap.
- Highlight pain points and corresponding design opportunities per stage.

## 6. Design Solution
- Trainee side:
  - Real-time HR ring and target zone status.
  - Equipment check-in and progress guidance.
  - Badge wall and pet-level progression.
- Coach side:
  - Monitoring dashboard for trainee state.
  - Strategy push (pace up / steady fat burn / recovery).
  - Temporary motivation buff (double XP).

## 7. Prototype & Implementation
- Platform: Web app (HTML/CSS/JavaScript).
- Demo URL: https://rainm1st.github.io/
- Data sync approach: localStorage cross-tab synchronization for demonstration.

## 8. Preliminary Evaluation (Week 8)
- Metrics:
  - Task success rate
  - Task completion time
  - SUS score (quick usability measure)
- Initial insights:
  - Realtime cues reduce uncertainty during training.
  - One-tap coach actions fit onsite pace.

## 9. Novelty & Significance
- Combines realtime intensity feedback, coach intervention, and gamified continuity in one lightweight web workflow.
- Focuses on onsite interaction quality rather than only post-hoc activity tracking.

## 10. Next Steps (to Week 10)
- Improve responsiveness and UI clarity.
- Run broader usability tests (6-10 users).
- Add quality-focused recap metrics and finalize 2-minute demo video.
